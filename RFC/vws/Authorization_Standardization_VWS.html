Authorization Standardization

Dutch Healthcare Profile (AuthZEN-based)

Summary

Within the Dutch healthcare domain, authorization is positioned as a generieke functie (generic function).

  • operate system-wide across healthcare domains
  • be independent of individual applications
  • be standardizable and normeerbaar
  • support interoperability

This RFC defines a standardized authorization contract based on:

  • OpenID AuthZEN Authorization API 1.0
  • NLGov AuthZEN profile

The contract is PDP-agnostic: it standardizes the PEP–PDP interaction at the protocol level, without prescribing how policies are authored or evaluated. Any engine exposing an AuthZEN-compliant decision endpoint — ABAC, RBAC, ReBAC, or hybrid, built on OPA, Cedar, OpenFGA, XACML, or custom — can be plugged in without changes to consuming applications, allowing organizations to choose their PDP while preserving a uniform authorization interface.

Architecture Overview

The Preprocessor has no authorization intelligence of its own. Its sole responsibility is to translate an incoming request into a well-formed AuthZEN authorization request — populating the subject, action, resource, and context fields — so that the PDP can interpret and evaluate it. All policy logic resides exclusively in the PDP.

The exact content of the AuthZEN request depends on the domain (iWlz, Zvw, Wmo, Jeugdwet — see Scope). Each domain-specific RFC defines which attributes must be present and how they are populated. This document establishes the generic parent contract; domain RFCs inherit and extend it without deviating from the base structure defined here.

Scope

This RFC defines the generic authorization contract for the Dutch healthcare domain. It is the normative parent specification. Domain-specific RFCs (e.g. iWlz, Zvw, Wmo, Jeugdwet) inherit this contract and may only extend it — they may not deviate from or replace the base contract defined here.

The following domains fall within scope:

Domain Description Source
iWlz Information provision for long-term care (Wlz). Authorization governs which organizations may access or mutate client care records based on their role (zorgkantoor, zorgaanbieder, CIZ, CAK) and regional or contractual relationship. iWlz RFC repository
Zvw Basic health insurance (Zorgverzekeringswet). Authorization governs access to declaratie, vergoeding, and verzekeringsdata based on the insurer–provider relationship and population ownership. Vektis standaardisatie
Wmo Social support act (Wet maatschappelijke ondersteuning 2015). Authorization governs municipality access to citizen support decisions and provider access to iWmo messages within active municipal contracts. iStandaarden iWmo
Jeugdwet Youth care act. Authorization governs municipality access to youth care decisions and youth care provider access to iJw messages within active assignments. iStandaarden iJw

Out of scope

  • Authentication and identity proofing — governed by NLGov OAuth/OIDC profiles.
  • Policy authoring and evaluation logic inside the PDP — only the PEP–PDP interface is standardized here.
  • Patient/citizen consent management — addressed separately under MITZ/NUTS.
